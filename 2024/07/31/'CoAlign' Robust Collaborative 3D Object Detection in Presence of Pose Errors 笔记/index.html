CoAlign 笔记

发表于2024-07-31|更新于2024-07-31|BEV
|字数总计:1.2k|阅读时长:5分钟

Intro

  • 多Agent协同BEV中，多车要同步位姿到一个统一的坐标种
  • 6DoF的位姿带来相对位姿误差，降低协作质量
    • Learning to communicate and correct pose errors (V2VNet)使用位姿回归预测模块
    • Keypoints-based deep feature fusion for cooperative vehicle detection of autonomous driving 使用3D点语义标签对齐点级的对应关系
    • ⬆️都需要监督学习（提供GT的位姿）
  • CoAlign
    • 多Agent共享中间特征和单个Agent的检测结果
    • 用多Agent检测结果纠正定位误差

Related Works

  • 协作感知
    • 数据集 V2X-Sim OPV2V Comap DAIR-V2X
    • DiscoNet ：训练阶段提出了一种师生知识蒸馏框架，以利用早期融合中更全面的感知信息
    • SyncNet
  • 协作感知中的精确定位
    • 监督学习的方法
      • Learning to communicate and correct pose errors （V2VNet） 使用位姿回归预测、注意力机制，纠正相对位姿
      • Overcoming obstructions via bandwidth-limited multi-agent spatial handshaking （MASH）
    • 设计强大的网络架构
      • V2x-vit: Vehicle-to-everything cooperative perception with vision transformer 多级窗口注意力
      • Keypoints-based deep feature fusion for cooperative vehicle detection of autonomous driving （FPV-RCNN）
    • ⬆️需要GT监督
  • SLAM
    • 基于图的SLAM
      • Globally consistent range scan alignment for environment mapping
      • A tutorial on graph-based slam

Problem Formulation

  • Collaboration Mode

    • Early Fusion
      • Cooperative perception for 3d object detection in driving scenarios using infrastructure sensors
      • Opv2v: An open benchmark dataset and fusion pipeline for perception with vehicle-to-vehicle communication
    • Intermediate Fusion
      • V2xsim: Multi-agent collaborative perception dataset and benchmark for autonomous driving
      • V2x-vit: Vehicle-to-everything cooperative perception with vision transformer
    • Late Fusion
      • Cooperative perception for 3d object detection in driving scenarios using infrastructure sensors
      • Opv2v: An open benchmark dataset and fusion pipeline for perception with vehicle-to-vehicle communication

  • Formulation

    • ξ=(xi,yi,zi,θi,ϕi,ψi)\xi =(x_i, y_i, z_i, \theta_i, \phi_i, \psi_i)ξ=(xi​,yi​,zi​,θi​,ϕi​,ψi​)
    • 6DoF有噪声，从不同agents接收到的特征在经过位姿变换之后会有不同的坐标系，无法实现空间对齐，导致特征融合错位，进一步使得目标检测不准确

Method

  • CoAlign Architecture

    • 2a部分在单个Agent上提取特征
    • 2b基于所有检测结果纠正位姿误差
    • 2c根据校正后的姿势同步其他Agent的特征
    • 2d特征融合 2e解码预测

  • 单个Agent检测

    • 使用PointPillars
    • 估计不确定性： 3D bounding box的中心方位、长宽高、yaw角、中心位置方差和角度方差
    • 中心位置Loss Lx=(x^−x0)2/(2σx2)+log⁡(σx2)/2L_x=(\hat x-x_0)^2/(2\sigma^2_x)+\log(\sigma^2_x)/2Lx​=(x^−x0​)2/(2σx2​)+log(σx2​)/2
    • yaw角Loss TODO
    • 总Loss L=LCrossEntropy+αregLsmooth−L1+αcenter(Lx+Ly)+αθLθL=L_{CrossEntropy}+\alpha_{reg} L_{smooth-L1}+\alpha_{center}(L_x+L_y)+\alpha_\theta L_\thetaL=LCrossEntropy​+αreg​Lsmooth−L1​+αcenter​(Lx​+Ly​)+αθ​Lθ​

  • 相对位姿纠正

    • 为了对齐各Agent的相对位姿，需要对齐同一物体的Bounding Box

    • 利用Agent-对象姿态 图 表示Agent和对象之间的关系

    • 第i个Agent接受来自所有其他代理的消息并构建内部Agent-对象姿态图G(Vagent,Vobject,E)G(\mathcal V^{\mathrm{agent}}, \mathcal V^{\mathrm{object}}, \mathcal {\Epsilon})G(Vagent,Vobject,E)，这是一个二部图

    • Vobject\mathcal V^{\mathrm{object}}Vobject 是由所有Agent的bbox空间聚类得到的

    • Note zjk=(x^,y^,θ^)z_{jk}=(\hat x, \hat y, \hat \theta)zjk​=(x^,y^​,θ^) 为Agent j看物体k的姿态

    • 理想情况下物体姿态从多个智能体看是一致的

    • Note ξj\xi_jξj​ 为Agent j 的Ego姿态，χk\chi_kχk​ 为物体k的Ego姿态，⊙\odot⊙ 为对应的齐次坐标矩阵相乘

    • 那么观测误差就写成ejk=zjk−1⊙(ξj−1⊙χk)\mathbf e_{jk}=z_{jk}^{-1}\odot (\xi_j^{-1}\odot \chi_k)ejk​=zjk−1​⊙(ξj−1​⊙χk​)

    • 消除位姿估计误差的优化问题就写成

    • {ξj∗,χk∗}=arg⁡min⁡{ξj,χk}∑(j,k)∈EejkTΩjkejk\{\xi_j^*, \chi_k^*\} = \arg \min_{\{\xi_j, \chi_k\}} \sum_{(j,k) \in \mathcal{E}} e_{jk}^T \Omega_{jk} e_{jk}{ξj∗​,χk∗​}=argmin{ξj​,χk​}​∑(j,k)∈E​ejkT​Ωjk​ejk​，where Ωjk=diag(1σx2,1σy2,1σθ2)\Omega_{jk} = \text{diag}\left(\frac{1}{\sigma_x^2}, \frac{1}{\sigma_y^2}, \frac{1}{\sigma_\theta^2}\right)Ωjk​=diag(σx2​1​,σy2​1​,σθ2​1​) 为信息矩阵，代表各类误差的权重；这个表达式是姿态误差的加权平方和

  • 多尺度特征融合

    • 这部分和HEAL里的很像
    • Fj→i(1)=Mj→i,j=1,2,…,N,Fj→i(ℓ+1)=gℓ(Fj→i(ℓ)),ℓ=1,2,…,L,Fi(ℓ)=Fuse({Fj→i(ℓ)}j=1,2,…,N),Fi′=Cat([Fi(1),u2(Fi(2)),…,uL(Fi(L))]),\begin{aligned}& \mathbf{F}^{(1)}_{j \to i} = \mathbf{M}_{j \to i}, \quad j = 1, 2, \ldots, N, \\& \mathbf{F}^{(\ell+1)}_{j \to i} = g_{\ell}(\mathbf{F}^{(\ell)}_{j \to i}), \quad \ell = 1, 2, \ldots, L, \\& \mathbf{F}^{(\ell)}_{i} = \text{Fuse}(\{\mathbf{F}^{(\ell)}_{j \to i}\}_{j=1,2,\ldots,N}), \\& \mathbf{F}'_{i} = \text{Cat}([\mathbf{F}^{(1)}_{i}, u_{2}(\mathbf{F}^{(2)}_{i}), \ldots, u_{L}(\mathbf{F}^{(L)}_{i})]),\end{aligned}​Fj→i(1)​=Mj→i​,j=1,2,…,N,Fj→i(ℓ+1)​=gℓ​(Fj→i(ℓ)​),ℓ=1,2,…,L,Fi(ℓ)​=Fuse({Fj→i(ℓ)​}j=1,2,…,N​),Fi′​=Cat([Fi(1)​,u2​(Fi(2)​),…,uL​(Fi(L)​)]),​
    • 最后的Fi′\mathbf F_i^{'}Fi′​ 送给解码端输出预测
