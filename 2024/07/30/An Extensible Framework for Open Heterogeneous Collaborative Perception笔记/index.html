HEAL 笔记

发表于2024-07-30|更新于2024-07-30|BEV
|字数总计:479|阅读时长:1分钟

Intro

  • 先前的工作都默认了所有agent都是一样的，共用一个感知处理模型
  • 解决多种异构传感器的方法
    1. Late fusion：将各种传感器的感知输出融合，易受到定位噪声和通信延迟的影响
    2. 收集所有感知输入 Collectively训练，增加新种类agent带来的代价大
  • HEAL
    • 保持一个一致恒定的特征空间
    • 新种agent加入时，固定特征检测后端，只训练更新新agent的编码端
  • OPV2V-H数据集

Methods

  • Open Heterogeneous Collaborative Perceptron

    • Agents类型不完全确定，Agent结构和传感器模式可以逐渐增加

  • HEAL

    • Collaborative Base Training

      • 使用一些初始的传感器数据训练一个特征融合、解码端网络
      • 某种结构j的输入数据先经过自己的encoder端提取特征图
      • j的特征图传输、空间转换给i
      • i特征融合网络输出检测结果

    • Pyramid Fusion

      • 多层下采样过ResNet 再上采样累加特征图
      • 优化目标
        L=Ldet(Bi,Yi)+∑l∑jαlLfocal(Sj→i(l),Yj→il)L=L_{\mathrm{det}}(\mathbf{B}_i,\mathbf{Y}_i)+\sum _l\sum_j \alpha_lL_{focal}(\mathbf{S}^{(l)}_{j\to i} ,\mathbf{Y}^{l}_{j\to i})L=Ldet​(Bi​,Yi​)+∑l​∑j​αl​Lfocal​(Sj→i(l)​,Yj→il​)
        第一部分是检测端误差，B是输出检测图，Y是GT；使用了focal loss和**smooth-L1 损失**
        第二部分是金字塔每层的损失，由预测前景图和BEV掩码GT计算focal loss
        α\alphaα是控制权重的超参

    • New Agent Type Training

      • 保持Pyramid Fusion特征空间参数不变，只训练新Agent编码器部分
      • 仅对新型的单个代理进行培训。这显着降低了每次集成的训练成本

Experiments

  • Datasets
    • OPV2V-H
    • DAIR-V2X
  • Details
    • LiDAR数据 PointPillars 作为基础模型
    • RGB数据 Lift-splat(LSS) with EfficientNet/ResNet50模型 、LiDAR SECOND模型，三种模式分别作为new agent types
